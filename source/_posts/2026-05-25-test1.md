# CWFM-Net (Causal Wavelet Flow-Matching Network)

## 实验思路

CWFM-Net 是一个**条件 Flow Matching** 图像复原框架。核心思想：学习一个速度场 v_θ，通过 10 步 ODE 积分将退化图像"流"向干净图像。

### Pipeline 概览

```
X_deg (退化图像)
  → [Stage 1] 冻结 BaryIR-5D encoder → z_bary (语义不变锚点)
  → [Stage 2] DAMOmegaEstimator → omega (退化频率分布 [B,4])
  → [Stage 3] DWT → WaveletPromptBlock → IDWT → P_cond (频率条件)
  → [Stage 4] GreenVelocityField(cat[x_t, P_cond], t, z_bary) → v (速度)
              ├─ AGF1 门控 skip e0 (过滤退化残留)
              └─ AGF2 门控 skip e1 (过滤退化残留)
  → [Stage 5] 训练: MSE(v_pred, v_target) + λ·L_freq
  → [Stage 6] 推理: 10步 Euler ODE (t: 1→0) → Y_restored
```

### 各 Stage 详解

- **Stage 1**: 加载预训练 BaryIR-5D，冻结参数。提取 z_bary（Wasserstein Barycenter 空间的 bottleneck），作为退化无关的语义锚点。
- **Stage 2**: 多尺度 depthwise conv + 空间门控 + 双统计量池化，端到端隐式学习退化在 4 个小波子带的能量分布 omega（无独立自监督损失，梯度从 flow matching MSE 经 P_cond → WPB → omega 回传）。
- **Stage 3**: 对 X_deg 做 Haar DWT，每个子带用可学习 prompt 做 SFT 调制（受 omega 门控），再 IDWT 回空间域得到 P_cond。
- **Stage 4**: Green 函数速度场 UNet。
  - 输入: cat[x_t, P_cond] = 6 通道；输出: 3 通道速度（RGB 空间）。VF 初始化为 in_channels=6，out_conv 被覆盖为 Conv2d(base_dim, 3, 1) 以匹配输出维度。
  - 编码器/解码器只受时间条件调制（AdaGN）。
  - 瓶颈层用 z_bary 偏置 Q 向量实现退化感知注意力（Green 函数核）。
  - **AGF (Adaptive Gated Fusion)**: 通过 `_patch_vf_with_agf()` 注入 VF 的 skip connection。agf2 门控 e1（H/2 尺度），agf1 门控 e0（全分辨率），用空间+通道双门控过滤编码器传来的退化残留。
- **Stage 5**: Rectified Flow 训练。
  - 时间采样: t ∈ [0.01, 0.99]（边界裁剪避免 t=0/1 数值不稳定）。
  - 插值: x_t = (1-t)·clean + t·start，v_target = clean - start。
  - 学习 v_θ 逼近 v_target，v_pred 指向 clean 方向。
- **Stage 6**: 10 步 Euler 积分从 t=1 到 t=0。dt 为负值，更新公式 x = x - dt·v 等效于 x += |dt|·v，沿 v（指向 clean）方向积分。输出复原图像，生成对比图和评估指标。

### 扩展/备选方案 (stage4)

`green_velocity_field.py` 中包含以下实验性变体，当前主流程未使用：

- **DegradationEncoder (Plan A)**: 从 x_t 提取 128d 退化嵌入 res_emb，捕获任务特定退化特征。
- **GreenBottleneckAttnV2**: Q-bias 额外接收 res_emb（cond = cat[t_emb, b_emb, res_emb]），让注意力核感知具体退化类型。
- **GreenVelocityFieldV2 (Plan A + Plan B)**:
  - Plan A: 加 DegradationEncoder，res_emb 注入 bottleneck Q-bias。
  - Plan B: 解码器末端加空间门 M ∈ [0,1]^{H×W}（初始化 sigmoid(4)≈0.982，近似全开），逐像素控制速度幅度。

## 已知性能问题

**z_bary 重复计算**: `trainer.py` 训练循环中 `net.bary_enc.forward(x_deg)` 先算了 z_bary，随后 `net.encode_condition(x_deg)` 内部又调用 `self.bary_enc.get_z_bary(x)` 跑了第二遍 BaryIR encoder。`evaluator.py` 的 `sample_rf()` 同样存在此问题。应将 `encode_condition` 改为接受外部传入的 z_bary 以避免冗余前向。

### 6. Optuna 自动超参搜索

> 需安装: `pip install optuna`

```bash

python tune_optuna.py --bary_ckpt checkpoint/model_allBaryIR__5D.pt --tasks dehaze --n_trials 50

# 多任务搜索，缩短每次 trial 训练时间
python tune_optuna.py --bary_ckpt checkpoint/model_allBaryIR__5D.pt --tasks dehaze,derain --n_trials 30 --epochs 8

# 从中断的 study 继续搜索 (需指定相同的 study_name)
python tune_optuna.py --bary_ckpt checkpoint/model_allBaryIR__5D.pt --tasks dehaze --n_trials 20 --study_name cwfm_dehaze_0524_1030 --resume

# 用 Hyperband 剪枝 (更激进的早停)
python tune_optuna.py --bary_ckpt checkpoint/model_allBaryIR__5D.pt --tasks dehaze --n_trials 100 --pruner hyperband

# 搜索完毕后用最优参数正式训练 (脚本会自动打印命令)
python tune.py --bary_ckpt checkpoint/model_allBaryIR__5D.pt --tasks dehaze --lr 0.000182 --lambda_freq 0.087 --base_dim 48 --epochs 15
```

## 参数调优优先级建议

**第一梯队（对性能影响最大）:**
1. `--lr` — 学习率是最敏感的超参
2. `--tasks` — 任务组合决定泛化能力
3. `--residual_mode` — 改变 flow 路径长度，对收敛速度影响大
4. `--base_dim` — 模型容量 vs 显存的核心权衡

**第二梯队（精调阶段）:**
5. `--lambda_freq` — 频率 loss 对高频细节恢复有帮助
6. `--patch_size` — 更大 patch 看到更多上下文，但显存翻倍
7. `--attn_heads` — 注意力粒度
8. `--n_steps` — 推理质量 vs 速度

**第三梯队（需要改代码的硬编码参数）:**
9. `hidden` / `scales` (Stage 2) — omega 估计器容量
10. `n_prompts` / `prompt_ch` (Stage 3) — 小波 prompt 表达力
11. `t_emb_dim` (Stage 4) — 时间嵌入维度
12. `lf_boost` / `cutoff` (freq_loss) — 频率 loss 形状
