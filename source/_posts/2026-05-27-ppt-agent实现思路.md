# PPT-Agent报告

## 一、核心设计理念

### 核心思想：内容驱动，而非模板驱动

传统 PPT 生成工具的问题：
- 直接从主题跳到成品页面
- 用固定模板填充内容
- 缺少中间审核环节
- 生成的内容空洞、缺乏逻辑

PPT-Agent 的解决方案：
- 分阶段渐进式生成
- 每个阶段都有明确的交付物
- 用户可以在关键节点介入审核
- 内容驱动布局选择，而非模板驱动内容

### 工作流设计：模拟专业团队

PPT-Agent 的工作流模拟了一个专业演示文稿团队的协作过程：

需求澄清 → 资料调研 → 大纲规划 → 策划稿 → 设计稿 → 复核

## 二、五大核心阶段详解

### Stage 1: 访谈阶段

**目标**：通过多轮对话，深度理解用户需求，生成调研底稿

### 工作流程

- Step 1.0: 广度搜索
- Step 1.1: 第一轮追问（锁定场景与受众）
- Step 1.2: 深度搜索
- Step 1.3: 第二轮追问（挖掘核心亮点）
- Step 1.4: 第三轮追问（框定边界与约束）
- Step 1.5: 生成调研底稿

### 各步骤的 Prompt 设计

**Step 1.0: 广度搜索**

输入：用户提供的主题

输出 JSON：

```json
{
  "topic_summary": "主题一句话概括",
  "typical_scenes": ["场景1", "场景2"],
  "typical_audiences": ["受众1", "受众2"],
  "common_sections": ["板块1", "板块2"],
  "typical_styles": ["风格1", "风格2"]
}
```

---

**Step 1.1: 第一轮追问**

Prompt：

```
- 语气自然亲切，像真人对话，不要列表式提问
- 先简短回应用户的主题（表示你理解了），再抛出有引导性的问题
- 问题要具体，给出 2~3 个选项让用户容易回答
- 一次只问一个核心问题（可以附带一个小的补充问题）
- 禁止使用 markdown 格式符号（**、#、- 等）
```

---

**Step 1.2: 深度搜索**

输出 JSON：

```json
{
  "required_sections": [
    { "name": "板块名", "key_content": ["内容1"], "importance": "high/medium" }
  ],
  "key_metrics": ["指标1", "指标2"],
  "best_practices": ["做法1", "做法2"],
  "common_pitfalls": ["坑1", "坑2"]
}
```

---

**Step 1.3: 第二轮追问**

Prompt：

```
- 基于深度调研的标准框架，给出 2~3 个具体的亮点方向供用户选择
- 每个方向要具体到"比如..."的程度
- 每个方向的描述控制在 3~5 句话以内，简洁有力
```

---

**Step 1.4: 第三轮追问**

Prompt：

```
- 时间限制（汇报时长）
- 篇幅限制（页数）
- 素材情况（数据、图表、案例等）
- 特殊要求（必须包含/必须避免的内容）
```

---

**Step 1.5: 生成调研底稿**

输出 JSON 结构：

```json
{
  "research_brief": {
    "topic_summary": "主题一句话概括",
    "audience": "受众描述",
    "purpose": "演示目的",
    "scene": "具体场景",
    "time_limit": "时间限制",
    "page_count": "页数建议",
    "core_highlights": ["亮点1", "亮点2"],
    "key_facts": ["事实1", "事实2"],
    "recommended_sections": [
      { "title": "章节标题", "goal": "说明什么", "priority": "high/medium/low" }
    ],
    "available_materials": "用户手头素材描述",
    "risks_and_gaps": ["风险或缺口1"],
    "style_suggestion": "建议的表达风格"
  }
}
```

### 访谈阶段的独到之处

1. 搜索 → 追问 → 再搜索 → 再追问，不停的追问细节，摸清用户意图
2. 渐进式收集信息，从宽到窄，从模糊到具体
3. 通过选择题和引导性问题，为用户减负
4. 所有中间结果都是 JSON，便于后续阶段使用

---

### Stage 2: 大纲阶段

**目标**：基于调研底稿，生成层次清晰的 PPT 大纲

### Prompt 设计

引入金字塔原理：
1. 结论先行：每个部分先给核心观点
2. 以上统下：上层观点是下层内容的总结
3. 归类分组：同层内容必须属于同一逻辑范畴
4. 逻辑递进：按照时间、重要性、因果或并列关系组织

输出 JSON 结构：

```json
{
  "ppt_outline": {
    "cover": {
      "title": "主标题",
      "sub_title": "副标题"
    },
    "table_of_contents": {
      "title": "目录",
      "sections": ["第一部分标题", "第二部分标题"]
    },
    "parts": [
      {
        "part_title": "章节标题",
        "part_goal": "这一部分要说明什么",
        "pages": [
          {
            "page_number": 1,
            "title": "页面标题",
            "goal": "这一页的结论或作用",
            "key_points": ["要点1", "要点2"],
            "suggested_visual": "建议的视觉表达形式"
          }
        ]
      }
    ],
    "end_page": {
      "title": "总结与展望",
      "key_takeaways": ["核心结论1", "核心结论2"]
    },
    "total_pages": 8,
    "narrative_flow": "整体叙事逻辑一句话概括"
  }
}
```

### 大纲阶段的独到之处

1. 金字塔原理具体落实到 Prompt 中
2. 每页的 goal 可以明确每页的目的
3. 在大纲阶段就开始思考表达方式
4. 提供 refineOutline 接口，用户可以基于反馈调整

---

### Stage 3: 策划稿阶段

**目标**：把大纲转化为可供设计执行的策划稿

### 为什么需要策划稿？

传统工具的问题：
- 从大纲直接跳到设计稿
- 缺少”内容层级”和”表达方式”的中间层
- 设计师不知道哪些内容重要，哪些可以弱化

策划稿的价值：
- 明确每页的信息层级
- 指定推荐的视觉表达方式
- 提供证据和数据来源建议
- 给出设计注意事项

### Prompt 设计

每页策划卡包含的字段：

```json
{
  "page_number": 1,
  "title": "页面标题",
  "goal": "页面目标（这页最想让观众记住什么）",
  "core_messages": ["核心信息1", "核心信息2", "核心信息3"],
  "evidence_suggestions": ["数据来源1", "数据来源2"],
  "visual_type": "推荐表达方式",
  "layout_direction": "信息层级与布局方向描述",
  "keywords": ["关键词1", "关键词2"],
  "design_notes": "设计注意事项"
}
```

### 策划稿阶段的独到之处

1. 根据内容特点选择表达方式，而非套用固定模板
2. 明确哪些是主要信息，哪些是次要信息
3. 下一阶段的 AI 可以直接基于策划卡执行
4. 提供 refineOutline 接口，可以单独调整某一页

---

### Stage 4: 渲染阶段

**目标**：将策划卡转化为 1280×720px 的 HTML 演示页面

### Prompt 设计

**关键尺寸约束**：

```html
<!DOCTYPE html>
<html>
<head>
<style>
* { margin: 0; padding: 0; box-sizing: border-box; }
html, body { width: 1280px; height: 720px; overflow: hidden; }
.slide { width: 1280px; height: 720px; padding: 48px 56px; }
</style>
</head>
<body>
<div class="slide">
  <!-- 内容放这里 -->
</div>
</body>
</html>
```

---

### Stage 5: 导出阶段

将 HTML 页面导出为 PDF 或 HTML

## 三、核心创新点总结

1. **分阶段工作流**
    - 渐进式构建
    - 每个阶段都有明确的交付物和审核点
2. **策划稿中间层**
    - 填补了大纲到设计稿之间的空白
    - 明确了信息层级和表达方式
    - 让后续渲染更可控
3. **搜索-追问循环**
    - 动态收集信息
    - AI 先搜索建立直觉，再针对性提问
    - 降低了用户的负
4. **Prompt 设计**
    - 每个阶段都有专门的 Prompt
    - 明确的输入输出格式（JSON）
5. **Temperature 分层控制**
    - 搜索阶段：0.4-0.5（保证准确性）
    - 对话阶段：0.7-0.8（保证自然性）
    - 生成阶段：0.6
6. **渐进式披露**
    - 不一次性展示所有功能
    - 根据当前阶段显示相应的操作
    - 降低认知负担
7. **可视化可回退的进度**
    - 清晰展示当前进度
    - 每个阶段完成后有明确的反馈
    - 用户可以随时回到上一阶段修改自己的操作
8. **多层次修改**
    - 支持整体修改
    - 支持单页修改
    - 支持局部修改

## 四、改进方向

1. 最后输出的格式可编辑性太差，这就导致用户没有后期的修改空间，无法插入自己想要的图片等其他格式的文件
2. 尚不支持用户上传图片等不同格式的文件
3. 针对用户的身份，学习其偏好和风格，自动调整生成策略（这就意味着要准备多组prompt分别匹配不同的人群，然后用门控策略调节模型的输出）
4. …
